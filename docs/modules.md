# Описание модулей и функций safedrive

Документ описывает внутреннюю структуру алгоритма safedrive на уровне модулей и функций. Предназначен для разработчиков, которым потребуется править логику или добавлять новую функциональность. Концепции работы алгоритма с точки зрения пользователя см. в [userguide.md](../userguide/userguide.md).

## Архитектура

Алгоритм состоит из двух независимых PAWN-скриптов, которые на терминале Galileosky запускаются как разные EVM (виртуальные машины платформы):

- **`safedrive_main.p`** — фоновый цикл, запускается один раз при старте терминала и крутится непрерывно. На каждом тике опрашивает датчики и последовательно вызывает шесть функций модулей детекции.
- **`cmd_handler.p`** — одноразовый обработчик, запускается заново при каждой поступающей серверной команде, выполняет одну команду и завершается.

Скрипты обмениваются данными исключительно через разделяемые GSC-переменные платформы (доступны обоим через `GetVar/SetVar`). Их объявление вынесено в `safedrive.gsc`; в PAWN-коде они только используются. Часть состояния (счётчики перегревов, cold-start АКБ, настраиваемые параметры) дублируется в энергонезависимой памяти терминала через `ROMWrite/ROMRead`.

Модули детекции реализованы в виде файлов с расширением `.inc`, которые подключаются директивой `#include` в `safedrive_main.p`. Они не компилируются отдельно — препроцессор вставляет их содержимое в единый исходник перед компиляцией. Подключение в порядке зависимостей: сначала общие определения, затем хелперы, затем модули.

## Оформление

- Имена с префиксом `g` и заглавной буквой (`gCrashLastUtc`, `gWasParked`) — переменные. Файл-локальные `new` объявления на верхнем уровне `.inc` — внутреннее состояние модуля, видимое только внутри этого файла после препроцессинга. GSC-переменные из `safedrive.gsc` доступны через `GetVar/SetVar` обоим скриптам.
- Имена в `UPPER_CASE` — `#define`-константы из `safedrive_defs.inc`.
- Префиксы функций соответствуют модулю: `crashDetect`, `motionWatch`, `roadAccumulate`, `scoringTick`, `thermalWatch`, `batteryWatch`.

## safedrive_main.p

Главный фоновый скрипт. Подключает `safedrive_defs.inc`, `alert_sms.inc` и все шесть модулей детекции через `#include`. Содержит четыре функции.

- **`main()`** — точка входа. Выполняет `loadParams()`, `restoreState()`, публикует `TAG_RESTART=1` с принудительным `SavePoint`, после чего входит в бесконечный цикл с периодом `MAIN_LOOP_MS` (50 мс). На каждой итерации читает шесть величин (`UNIX_TIME`, акселерометр, `POWER`, `ENGINE_COOLANT_TEMPERATURE`, `SPEED`) с учётом симуляции, затем последовательно вызывает `crashDetect`, `motionWatch`, `roadAccumulate`, `scoringTick`, `thermalWatch`, `batteryWatch`, затем `processPendingCmd()`, и завершается вызовом `Delay(MAIN_LOOP_MS)`.
- **`processPendingCmd()`** — обработчик отложенных команд, которые обработчик команд не может выполнить сам (например, доступ к статическим переменным модулей). Читает `gCmdAction`, если в нём не `CMD_NONE` — вызывает соответствующую функцию (на текущий момент только `thermalResetCounters` для `CMD_RESET_OVERH`).
- **`loadParams()`** — загрузка параметров из трёх ROM-ключей с sanity-проверкой на старую раскладку. При первом запуске или при обнаружении некорректных значений (после миграции с предыдущей версии) — перезапись дефолтами из `DEF_*` констант. Преобразует пороги ускорения из mg в сырые единицы тега через `MG_TO_RAW`.
- **`restoreState()`** — восстановление долгоживущих счётчиков из ROM: `gOverheatCount`, `gOverheatTotalSec` (ключ `KEY_OVERHEAT`) и `gBatColdStartMv` (ключ `KEY_BATTERY`). Если ROM пуст, переменные остаются на дефолтных значениях из `safedrive.gsc`.

## cmd_handler.p

Одноразовый обработчик команд. Глобальные буферы (`gCmdText`, `gAnswer`, `gCmdName`, `gTmp`) объявлены на уровне скрипта и переживают вызовы пользовательских функций в пределах одного запуска. Поддерживает 15 команд, разнесённых по семи функциональным группам.

**Точка входа:**

- **`main()`** — диспетчер. Читает текст команды через `GetStringVar(COMMAND_TEXT, ...)`, выделяет имя команды до первого пробела через `extractCmdName`, и через цепочку `strcmp` вызывает соответствующий `handleXxx`.

**Команды оценки дороги:**

- **`handleStartRoadQ(connId, cmdNum)`** — проверяет, что сессия не активна; сбрасывает `gRoadSumAbsZ`, `gRoadSampleCount`, `gRoadPeakCount`; поднимает `gRoadActive`; возвращает «OK: roadq start».
- **`handleStopRoadQ(connId, cmdNum)`** — гасит флаг, проверяет минимум 10 отсчётов, считает MAD в mg через `RAW_TO_MG`, плотность пиков на 1000 отсчётов, выставляет балл по ступенчатой таблице (50/100/200/400 mg для MAD, 5/20/50/100 для peaks), публикует балл в `TAG_ROAD` с `SavePoint`, возвращает «ROAD=N mad=X peaks=Y».

**Команды скоринга:**

- **`handleStartScoring(connId, cmdNum)`** — проверка/сброс счётчиков `gScoringEpisodes`, `gScoringMs`, поднятие `gScoringActive`.
- **`handleStopScoring(connId, cmdNum)`** — гасит флаг, вызывает `calcDrivingScore`, публикует балл в `TAG_SCORE` с `SavePoint`.
- **`handleGetScore(connId, cmdNum)`** — возвращает промежуточный балл без остановки сессии.
- **`calcDrivingScore()`** — расчёт: `penalty = episodes * 10 + ms / 1000`, ступенчатая таблица (20/60/150/300 penalty → балл 4/3/2/1), возвращает int 1..5.
- **`replyScore(connId, cmdNum, const prefix[], score)`** — сборка ответа «PREFIX=N ep=M ms=K».

**Команды перегрева:**

- **`handleGetOverheat(connId, cmdNum)`** — формирует «OVERH cnt=N sec=K» из `gOverheatCount` и `gOverheatTotalSec`.
- **`handleResetOverheat(connId, cmdNum)`** — выставляет `gCmdAction = CMD_RESET_OVERH` для отложенного выполнения в главном цикле; возвращает «OK: overheat reset».

**Команда АКБ:**

- **`handleGetBatStat(connId, cmdNum)`** — читает `gBatColdStartMv`, если данных нет — возвращает «BAT no data»; иначе классифицирует по четырём диапазонам (используя `gV_BatOk`, `HC_V_BAT_LOW_MV`, `gV_BatCrit`) и формирует «BAT mv=K code=C».

**Команды параметров:**

- **`handleSetParam(connId, cmdNum)`** — парсит «x,y», валидирует индекс 1..`PARAM_COUNT`, вызывает `setParamValue` и `saveParamToRom`.
- **`handleGetParam(connId, cmdNum)`** — парсит индекс; при `x=0` возвращает все параметры через запятую, иначе один параметр.
- **`getParamValue(idx)`** — диспетчер по индексу, возвращает значение в пользовательских единицах (mg для порогов ускорения через `RAW_TO_MG`).
- **`setParamValue(idx, val)`** — диспетчер по индексу, конвертирует mg → raw через `MG_TO_RAW` для порогов ускорения.
- **`saveParamToRom(idx)`** — определяет, в какой из трёх ROM-ключей (`KEY_PARAMS_1..3`) попадает параметр, и перезаписывает весь ключ целиком.

**Команды SMS-номера:**

- **`handleSetSmsNum(connId, cmdNum)`** — посимвольно копирует аргумент в буфер длиной до `SMS_NUM_MAX_LEN` (15), пишет в `KEY_SMSNUM`.
- **`handleGetSmsNum(connId, cmdNum)`** — читает номер из ROM, возвращает «SMS=...» или «SMS not set».

**Команды симуляции:**

- **`handleSimAccel(connId, cmdNum)`** — парсит три значения в `gSimAx/gSimAy/gSimAz`, поднимает `gSimAccelEnabled`.
- **`handleSimEnv(connId, cmdNum)`** — парсит два значения в `gSimPowerMv` и `gSimCoolant` (sentinel-значения отключения: −1 и 32767 соответственно).
- **`handleSimOff(connId, cmdNum)`** — возврат всех симуляционных переменных к sentinel-значениям.

**Утилиты:**

- **`appendStr(const s[])` / `appendInt(value)`** — добавление к `gAnswer` через временный буфер `gTmp`.
- **`extractCmdName(const text{}, dest{}, maxLen)`** — копирует символы до пробела или нуля.
- **`findArgPos(const text{})`** — возвращает позицию первого символа аргумента после команды и пробелов, или −1.
- **`parseCSV(const text{}, startPos, values[], maxCount)`** — парсит до `maxCount` целых чисел через запятую, поддерживает знак минус, возвращает фактическое число прочитанных значений.
- **`reply(connId, cmdNum, const text[])`** — упаковывает литерал в `gAnswer` и отправляет.
- **`sendBuf(connId, cmdNum)`** — отправляет то, что уже собрано в `gAnswer`.

## safedrive_defs.inc

Файл с `#define`-директивами, не содержит исполняемого кода. Защищён от повторного включения парой `#if defined SAFEDRIVE_DEFS_INC / #endinput / #define`.

Группы определений:

- **Идентификаторы пользовательских тегов** (`TAG_CRASH`..`TAG_RESTART`, значения `0x62`..`0x69`) — используются в вызовах `TagWriteValue`.
- **Ключи ROM** (`KEY_PARAMS_1..3`, `KEY_OVERHEAT`, `KEY_BATTERY`, `KEY_SMSNUM`) — идентификаторы записей в энергонезависимой памяти.
- **Индексы параметров** (`PARAM_A_CRASH`..`PARAM_T_BAT_RESET_SEC`, 1..12) — для маршрутизации в `setParamValue`/`getParamValue` обработчика команд.
- **Дефолты** (`DEF_*`) — значения параметров по умолчанию при пустом ROM. Пороги ускорения в mg, времена в секундах, напряжения в милливольтах, температуры в °C.
- **Захардкоженные пороги** (`HC_*`) — пороги, не выводимые в SETSDPARAM ради экономии GSC-глобалок (лимит ~30 на программу): `HC_V_TOW_MIN`, `HC_T_TOW_LOCKOUT`, `HC_V_BAT_LOW_MV`, `HC_V_ENGINE_ON_MV`.
- **Макросы конверсии единиц** — `MG_TO_RAW(x)` и `RAW_TO_MG(x)`, конверсия между mg и сырыми единицами тега акселерометра (1 G = 8192 единиц).
- **Константа гравитации** — `GRAVITY_RAW` (8192), вычитается из оси Z в `motion_watch` для получения «избыточного» ускорения.
- **Коды отложенных команд** — `CMD_NONE`, `CMD_RESET_OVERH` (передаются через GSC-переменную `gCmdAction`).
- **Параметры цикла** — `MAIN_LOOP_MS` (50 мс), `BASELINE_INIT_SAMPLES` (20 отсчётов для инициализации базовой линии в road_quality), `ROAD_SAMPLE_CAP` (20000, верхняя граница накопления).
- **Параметры SMS** — `SMS_NUM_MAX_LEN` (15 символов), `SMS_BUF_SIZE` (128 байт).

## alert_sms.inc

Общий хелпер для отправки SMS-оповещений, используется из `crash_detector.inc` и `motion_watch.inc`. Защищён от повторного включения.

- **`isqrt(n)`** — целочисленный квадратный корень методом Ньютона. Принимает `int`, возвращает floor(√n). Граничные случаи: `n ≤ 0` → 0, `n < 2` → n. Сходимость квадратичная: для n до 10⁹ хватает 10–15 итераций.
- **`calcMagGx10(sumSqHalved)`** — конверсия суммы квадратов «уполовиненных» компонент акселерометра в модуль ускорения в десятых долях G. Истинный |a| = 2 · √(sumSqHalved); далее перевод сырых единиц в G·10 через деление на 819.2. Возвращает целое: 36 = 3.6 G.
- **`sendAlertSms(const event[], magGx10)`** — основная функция. Читает номер из `KEY_SMSNUM`; если пуст, пишет «SMS skip, no number» в диагностику и выходит. Иначе через нативные `strpack`/`strcat`/`valstr` собирает строку формата «SendSMS <phone>, <EVENT> a=<X.Y>G %IMEI %LAT %LON», передаёт в `ExecCommand`. Плейсхолдеры `%IMEI`, `%LAT`, `%LON` подставляет уже сам терминал перед отправкой.

## crash_detector.inc

Детектор ДТП.

Состояние: `gCrashLastUtc` (таймштамп последнего срабатывания для антидребезга).

- **`crashDetect(ax, ay, az, nowUtc)`** — единственная функция модуля. Проверяет антидребезг по `gT_CrashLockout`. Считает sum-of-squares уполовиненных компонент (защита от переполнения int32) и сравнивает с квадратом порога `gA_Crash/2`. При превышении: фиксирует `gCrashLastUtc`, публикует `TAG_CRASH=1`, форсирует `SavePoint`, пишет в диагностику «SD: CRASH! a2=… thr2=…», вызывает `sendAlertSms("CRASH", calcMagGx10(sumSq))`.

## motion_watch.inc

Детектор несанкционированного перемещения припаркованного автомобиля.

Состояние:
- `gParkedStartUtc` — момент, когда условия «припаркован» начали выполняться.
- `gWasParked` — sticky-флаг «был припаркован» (1 после удержания условий ≥ `gT_ParkedMin` секунд, сбрасывается только на фронте запуска двигателя).
- `gPrevEngineOnAtMotion` — предыдущее значение `engineOn` для детекции фронта.
- `gTowLastUtc` — таймштамп последнего срабатывания для антидребезга.

- **`motionWatch(ax, ay, az, gpsSpeed, powerMv, nowUtc)`** — основной обработчик. Вычисляет `engineOn` как `POWER ≥ HC_V_ENGINE_ON_MV`. На фронте запуска двигателя сбрасывает sticky-флаг и таймер. Управляет накоплением «припаркован»-состояния. После проверки антидребезга считает `sumSq` уполовиненных компонент с **вычетом GRAVITY_RAW из az** (чтобы порог сравнивался с избыточным ускорением, а не с полным модулем, включающим 1 G). Проверяет два триггера: (1) одиночный спайк выше `gA_Motion`, (2) GPS-движение выше `HC_V_TOW_MIN` км/ч при заглушённом двигателе.
- **`motionTrigger(nowUtc, reason, sumSqHalved)`** — фиксирует `gTowLastUtc`, публикует `TAG_TOW=1` с форсированной отправкой, пишет «SD: TOW! reason=N» в диагностику, вызывает `sendAlertSms("TOW", ...)`.

## road_quality.inc

Накопление статистики покрытия дороги в сессионном режиме.

Состояние:
- `gRoadBaselineInited` — флаг завершения инициализации базовой линии.
- `gRoadBaselineSum` — сумма для усреднения на первых `BASELINE_INIT_SAMPLES` отсчётах.
- `gRoadBaseline` — текущая базовая линия (медленный фильтр гравитации).

- **`roadAccumulate(az)`** — единственная функция. Проверяет `gRoadActive` (выставляется обработчиком команд при `STARTROADQ`); при сбросе сессии обнуляет внутреннее состояние. Первые 20 отсчётов усредняет в базовую линию. Дальше использует однополюсный IIR-фильтр с τ ≈ 13 сек (`gRoadBaseline += (az - gRoadBaseline) / 256`) для отслеживания медленных изменений ориентации. Считает `delta = az - gRoadBaseline` (отклонение от равновесия), накапливает в `gRoadSumAbsZ` (с защитой от переполнения через `cellmax`), увеличивает `gRoadSampleCount` до `ROAD_SAMPLE_CAP`. Если `|delta| > gA_RoadBumpThr` — инкрементирует `gRoadPeakCount`. Итоговый балл считается в обработчике команд при `STOPROADQ`.

## driving_score.inc

Скоринг стиля вождения в сессионном режиме.

Состояние:
- `gAccActive` / `gBrkActive` / `gTurnActive` — sticky-флаги активности каждого из трёх типов событий (для детекции фронтов).

- **`scoringTick(ax, ay)`** — единственная функция. Проверяет `gScoringActive` (выставляется при `STARTSCORING`); при сбросе — возвращается. Проверяет три условия независимо: `ay > gA_AccHard`, `ay < -gA_BrkHard`, `|ax| > gA_TurnHard`. Для каждого по фронту нарастания (sticky-флаг переходит из 0 в 1) увеличивает `gScoringEpisodes` на 1. Пока хотя бы одно условие активно — увеличивает `gScoringMs` на `MAIN_LOOP_MS` (50 мс). Итоговый балл считается в `calcDrivingScore` обработчика команд.

## thermal_watch.inc

Контроль перегревов двигателя.

Состояние:
- `gIsOverheatActive` — флаг текущего состояния «в перегреве».
- `gOverheatStartTime` — таймштамп начала текущего перегрева (для возможного восстановления после ребута, на данный момент не используется в логике).
- `gThermalLastSecUtc` — предыдущий таймштамп для подсчёта прироста суммарного времени.

- **`thermalWatch(coolantTemp, nowUtc)`** — конечный автомат. На фронте превышения порога `gT_OverheatThr`: инкремент `gOverheatCount`, выставление активного флага, публикация `TAG_OVERHEAT=1` и `TAG_OVERHEAT_REPEATED=count`, `SavePoint`, сохранение в ROM. В фазе удержания: накопление `gOverheatTotalSec` на разницу между текущим и предыдущим тиком. На фронте спада: гашение флага, сохранение в ROM.
- **`thermalSaveRom()`** — сохранение `gOverheatCount` и `gOverheatTotalSec` в `KEY_OVERHEAT`.
- **`thermalResetCounters()`** — вызывается из `processPendingCmd` по `CMD_RESET_OVERH`. Обнуляет счётчики в RAM и в ROM, сбрасывает локальное состояние.
- **`thermalIsActive()`** — геттер для `gIsOverheatActive`, оставлен для возможного использования другими модулями.

## battery_watch.inc

Контроль состояния АКБ по «холодному» напряжению.

Состояние:
- `gEngineOffSinceUtc` — момент, когда `POWER` упал ниже `HC_V_ENGINE_ON_MV`.
- `gBatLastUpdateUtc` — таймштамп последнего обновления `gBatColdStartMv`.
- `gPrevEngineOnAtBat` — предыдущий `engineOn` для детекции фронта пуска.
- `gBatRomSaveUtc` — таймштамп последней записи в ROM (защита от частых записей).

- **`batteryWatch(powerMv, nowUtc)`** — основной обработчик. Поддерживает `gEngineOffSinceUtc` (сбрасывается при возврате к работе двигателя). Если двигатель устойчиво заглушён `gT_BatResetSec` секунд (по умолчанию 600) и с последнего обновления прошла минута — обновляет `gBatColdStartMv` текущим `powerMv` и пишет в ROM через `batterySaveRom`. На фронте запуска двигателя (если есть валидное `gBatColdStartMv > 0`): классифицирует через `batteryClassify`, публикует `TAG_BAT=code`, `SavePoint`, пишет в диагностику «SD: BAT start mv=… code=…».
- **`batteryClassify(mv)`** — возвращает 0..3 по четырём диапазонам (`gV_BatOk`, `HC_V_BAT_LOW_MV`, `gV_BatCrit`).
- **`batterySaveRom(nowUtc)`** — запись `gBatColdStartMv` и `nowUtc` в `KEY_BATTERY`. Не чаще раза в минуту.

## accel_dbg.p

Автономный диагностический скрипт, не связанный с основным алгоритмом. Используется на этапе монтажа терминала для проверки правильной ориентации осей акселерометра (см. раздел 5 [userguide.md](../userguide/userguide.md)).

- **`main()`** — точка входа. Бесконечный цикл с периодом 500 мс. На каждой итерации читает `ACC_X/Y/Z`, считает модуль вектора через `isqrt` (с защитой от переполнения через деление компонент на 2), переводит сырые единицы в милли-G (`mg = raw * 1000 / 8192`) и пишет в диагностику строку формата «ACC ax=… ay=… az=… |a|=… mg».
- **`isqrt(n)`** — локальная копия целочисленного корня методом Ньютона. Дублирует функцию из `alert_sms.inc`, поскольку `accel_dbg.p` компилируется как отдельная программа и не имеет доступа к модулям safedrive.
