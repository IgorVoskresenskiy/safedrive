# safedrive

Алгоритм комплексного мониторинга транспортного средства для терминала Galileosky 10 с использованием технологии EasyLogic и PAWN-скриптов.

## Возможности

- Детекция ДТП по модулю ускорения с SMS-оповещением на сохранённый номер.
- Детекция несанкционированного перемещения припаркованного автомобиля (буксировка, эвакуация, удар) с SMS-оповещением.
- Оценка качества дорожного покрытия по команде с сервера, балл 1–5.
- Скоринг стиля вождения по команде с сервера, балл 1–5.
- Контроль перегревов двигателя со счётчиком событий и накоплением суммарного времени в перегреве.
- Классификация состояния аккумуляторной батареи по напряжению покоя при каждом запуске двигателя.

## Структура репозитория

- `safedrive/` — основной алгоритм: главный цикл, обработчик команд и шесть модулей детекции.
- `accel_dbg/` — служебный скрипт для проверки ориентации осей акселерометра при первичной установке терминала.
- `binaries/` — скомпилированные бинарники для загрузки в терминал через раздел EasyLogic программы Configurator.
- `userguide/` — рукводоство пользователя системы.
- `docs/` — подробная документация по коду для разработчиков.

## Описание кода

Главный цикл `safedrive_main.p` опрашивает датчики терминала с периодом около 50 миллисекунд и последовательно вызывает функции шести независимых модулей детекции, каждый из которых отвечает за одну функцию мониторинга. Команды обслуживает отдельный одноразовый скрипт `cmd_handler.p`, запускаемый при поступлении каждой команды. Оба скрипта разделяют общие константы и идентификаторы тегов через подключаемый файл `safedrive_defs.inc`.

| Файл | Назначение |
|---|---|
| [`safedrive/safedrive_main.p`](safedrive/safedrive_main.p) | Главный цикл: чтение датчиков, последовательный вызов всех модулей, загрузка параметров из ROM и восстановление состояния после перезагрузки |
| [`safedrive/cmd_handler.p`](safedrive/cmd_handler.p) | Обработчик 15 серверных команд: оценка дороги, скоринг, перегревы, АКБ, параметры, SMS-номер, симуляция датчиков |
| [`safedrive/safedrive_defs.inc`](safedrive/safedrive_defs.inc) | Общие константы: идентификаторы пользовательских тегов, ключи ROM, индексы и значения параметров по умолчанию, макросы конверсии единиц |
| [`safedrive/crash_detector.inc`](safedrive/crash_detector.inc) | Детектор ДТП: одиночный спайк модуля ускорения выше порога A_CRASH с антидребезгом и SMS-оповещением |
| [`safedrive/motion_watch.inc`](safedrive/motion_watch.inc) | Детектор несанкционированного перемещения припаркованного автомобиля по триггерам акселерометра и GPS |
| [`safedrive/road_quality.inc`](safedrive/road_quality.inc) | Накопление статистики вертикальных колебаний по оси Z в сессионном режиме оценки дороги |
| [`safedrive/driving_score.inc`](safedrive/driving_score.inc) | Регистрация эпизодов агрессивных манёвров (разгон, торможение, поворот) в сессионном режиме скоринга |
| [`safedrive/thermal_watch.inc`](safedrive/thermal_watch.inc) | Контроль перегревов двигателя со счётчиками числа событий и суммарного времени в перегреве |
| [`safedrive/battery_watch.inc`](safedrive/battery_watch.inc) | Захват «холодного» напряжения АКБ во время простоя и классификация состояния при пуске двигателя |
| [`safedrive/alert_sms.inc`](safedrive/alert_sms.inc) | Общий хелпер для отправки SMS-оповещений с расчётом модуля ускорения в G |
| [`accel_dbg/accel_dbg.p`](accel_dbg/accel_dbg.p) | Диагностический скрипт вывода ускорений по трём осям в милли-G для калибровки ориентации терминала |

Конфигурация алгоритма (12 пороговых параметров) хранится в энергонезависимой памяти терминала и настраивается командой `SETSDPARAM`  без перепрошивки. Подробности — в руководстве пользователя.
